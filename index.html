2.3 Implementing Logic

Frame Rate Independence

Always multiply tick-based calculations by Delta Seconds to ensure consistent behavior across different frame rates!

2.3.3 The Stamina System

The stamina system prevents infinite sprinting and adds resource management to movement. It uses timers for regeneration delay and Event Tick for real-time updates.

// Stamina Drain Logic (Event Tick) if (IsSprinting && CurrentStamina > 0) { CurrentStamina -= StaminaDrainRate * DeltaSeconds; if (CurrentStamina <= 0) { StopSprinting(); } } else if (IsRegeneratingStamina && CurrentStamina < MaxStamina) { CurrentStamina += StaminaRegenRate * DeltaSeconds; }

3.2 Inventory Component

3.2.4 The Crucial Function: AddItem

The AddItem function handles item stacking and slot management. It must account for the Blueprint struct problem where array elements are copied by value.

The Struct Problem

When you get a struct from an array in Blueprints, you get a copy. Use Set Array Elem to modify the original array!

// AddItem Logic if (ItemIsStackable) { // Find existing stack for (Slot in InventorySlots) { if (Slot.ItemID == ItemID && Slot.Quantity < MaxStackSize) { // Update existing stack Slot.Quantity += AmountToAdd; SetArrayElement(InventorySlots, Index, Slot); break; } } }

4. Interaction System

4.1 The Interaction Interface

Interfaces define a contract for communication without requiring knowledge of specific object types. It's like a USB port standard - any device that implements it can communicate.

BPI_Interactable Functions

  • GetInteractionPrompt() → Returns display text
  • CanInteract() → Returns boolean permission
  • OnInteract(Interactor) → Executes interaction logic

4.2 Optimized Traces

4.2.1 Custom Trace Channel

A dedicated "InteractTrace" channel ensures traces only hit intended objects, improving performance and reliability.

Trace Configuration
  • Channel Name: "InteractTrace"
  • Default Response: Ignore
  • Interaction Distance: 300.0 units

4.3 Player Interaction Logic

4.3.2 The Interaction Trace

A line trace is performed every frame from the camera to detect interactive objects. The trace must ignore the player and only hit objects that block the InteractTrace channel.

// Trace Calculation Start = Camera->GetWorldLocation() Direction = Camera->GetForwardVector() End = Start + (Direction * InteractionDistance) // Perform Trace HitResult = LineTraceByChannel(Start, End, InteractTrace)

9. Bug Fixes and Polish

9.1 Debugging Techniques

Print String

Use for quick variable inspection and execution flow tracking.

PrintString("Stamina: " + CurrentStamina)

Breakpoints

Pause execution to inspect variable values by hovering over them.

Visual Debugging

Set Draw Debug Type to visualize traces and hit detection.

Common Issues & Solutions

  • Interaction Fails: Check collision settings and interface implementation
  • Struct Data Not Updating: Use Set Array Elem after modifications
  • T-Posing Meshes: Verify Leader Pose Component setup
  • Frame Rate Issues: Always multiply by Delta Seconds
